# 📊 Cube Documentation Tool

Интерактивный инструмент для управления документацией куба данных (OLAP куба). Позволяет каталогизировать меры, измерения, управлять связями между ними и составлять функциональные требования для развития куба.

## Функциональность

### 📖 Страница 1: Каталог куба
- Справка по всем мерам, измерениям и их связям
- Поиск и фильтрация по группам мер
- Наглядное отображение связей "мера → измерение"
- Информация о таблицах и выражениях DAX

### ✏️ Страница 2: Конструктор требований
- Создание функциональных требований для расширения куба
- Выбор между добавлением в существующую группу мер или созданием новой
- Добавление новых мер с выражениями DAX и форматированием
- Добавление новых измерений с описанием
- Управление связями между мерами и измерениями
- **Автоматическое добавление** существующих измерений при выборе группы
- Экспорт требований в **Markdown** и **DOCX** форматы
- Сохранение требований в браузер (localStorage)

## Установка и запуск

### Требования

### Сборка


## Тестовые данные

По умолчанию приложение содержит две группы мер для тестирования:

1. **F00002 - Планы первичных продаж**
   - Меры: plan_sale_kg, plan_next_month_kg, fill_fact_reserve
   - Связано с измерениями: D00001 (календарь), D00750 (SKU), D00800 (клиенты)

2. **F00006 - Скидки по первичным продажам**
   - Меры: fact_promo_kg, fact_promo_packs, fact_promo_rub_vat, etc.
   - Связано с измерениями: D00001, D00400 (скидки), D00551 (источник), D00750, D00800

## Как использовать

### Просмотр каталога
1. Перейдите на вкладку "Каталог куба"
2. Выберите группу мер из выпадающего списка
3. Используйте поиск для быстрого поиска мер
4. Просмотрите таблицы мер, измерений и связей

### Создание требования
1. Перейдите на вкладку "Требования"
2. Заполните название и описание требования
3. Выберите:
   - **Существующую группу мер** (чтобы добавить меры в неё)
   - **Создать новую группу** (если это новая группа мер)
4. При выборе существующей группы автоматически загружаются связанные измерения
5. Добавьте новые меры и/или измерения
6. Нажимите "⚡ Авто" чтобы автоматически создать связи
7. Или добавьте связи вручную используя сборщик справа
8. Нажмите "📄 Markdown" или "📋 DOCX" для скачивания
9. Нажмите "💾 Сохранить требование" для сохранения в браузер

### Экспорт
- **Markdown**: Подходит для совместной работы в Git
- **DOCX**: Полнофункциональный документ для отправки

Для парсинга вашего XML куба используйте функции из `mockData.js`:

```javascript
// Парсируйте XML и преобразуйте в объекты типов из types.js
import { Measure, Dimension, MeasureGroup } from './types';

// Затем обновите CUBE_DATA в mockData.js
```

## Сохранение данных

Все требования автоматически сохраняются в localStorage браузера под ключом `cube_docs_requirements`. Данные сохраняются при нажатии кнопки "Сохранить требование".
