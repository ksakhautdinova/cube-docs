# Дизайн приложения

## Назначение
Приложение проектируется как корпоративный инструмент для разработчиков баз данных и аналитической витрины. Интерфейс должен быть сдержанным, предсказуемым и близким по UX к рабочим паттернам `MS SQL`, `Visual Studio`, `VS Code`, `Airflow`.

## Основные принципы
1. Никаких эмодзи в интерфейсе, текстах кнопок, табах, заголовках и статусах.
2. Используется только один набор иконок во всем приложении: `lucide-react`.
3. Визуальная основа приложения ограничивается тремя базовыми цветами:
   - `#1F2937` — темная опорная поверхность (навигация, контрастные панели).
   - `#2563EB` — основной акцент (активные элементы, фокус, ссылки, выделения).
   - `#6B7280` — нейтральный служебный цвет (вторичный текст, разделители, подсказки).
4. Семантические состояния задаются теплыми стандартными цветами:
   - `Success`: `#16A34A`
   - `Warning`: `#F59E0B`
   - `Error`: `#DC2626`

## UX-ориентиры для корпоративного DB-инструмента
- Плотные, читаемые таблицы с четкой иерархией строк и заголовков.
- Контролы форм и фильтров в едином стиле (одинаковые высоты, отступы, состояния фокуса).
- Навигация без визуального шума: явное активное состояние раздела.
- Предсказуемые действия: `Сохранить`, `Экспорт`, `Добавить`, `Удалить` с понятной иконкой и подписью.
- Минимум декоративных эффектов, акцент на рабочем контенте и скорости восприятия.

## Тон и визуальный язык
- Стиль: корпоративный, инженерный, нейтральный.
- Типографика: системные UI-шрифты, умеренная контрастность, приоритет читаемости.
- Ритм: аккуратные сетки, единые радиусы, единые отступы, стабильные размеры полей.

## Ограничения реализации
- Новые компоненты и страницы обязаны использовать существующие CSS-токены темы.
- Цвета в стилях должны задаваться через переменные, без произвольных новых hex.
- Иконки добавляются только из `lucide-react`; смешивание наборов не допускается.
- Любая новая функциональность должна сохранять desktop-first UX для инженерных сценариев.
