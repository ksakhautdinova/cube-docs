# 👋 Cube Documentation Tool - Начните здесь

## ⚡ За 5 минут

### Что это?
Приложение для управления документацией OLAP куба - просмотр фактов, измерений, связей и создание требований.

### Быстрый запуск
```bash
cd cube-docs
npm install
npm run dev
```

---

## 📊 Две страницы

### 1. 📖 Каталог куба
- **Выбор сущности:** Факты (F00002, F00006) или Измерения (D00001, D00400, D00750)
- **Показатели (для F):** Название, Перевод, Описание, Выражение DAX
- **Атрибуты (для D):** Столбец, Перевод, Описание, Тип данных
- **Связи:** Какие таблицы связаны и по каким полям
- **Поиск:** Быстрый поиск по названиям
- **Data Lineage:** Вкладка для DBT/Git (в разработке)

### 2. ✏️ Требования
- Создание функциональных требований
- Добавление новых мер и измерений
- Управление связями (авто + ручное)
- Экспорт в DOCX или Markdown
- Сохранение в браузер

## 🔧 Настройка данных

### Добавление своих данных

Отредактируйте `cube-docs/src/mockData.js`:

**Для добавления факта:**
```javascript
const F00099_GROUP = new MeasureGroup(
  'F00099',
  'F00099 Ваша таблица фактов',
  'Описание',
  'f00099_table_name',
  [
    {
      id: 'M999',
      name: 'your_measure',
      translation: 'Ваша мера',
      description: 'Описание меры',
      expression: 'SUM([field])',
      formatString: '#,0.00'
    }
  ],
  ['D00001'] // связанные измерения
);
```

**Для добавления измерения:**
```javascript
new DimensionEntity(
  'D00999',
  'D00999 Ваше измерение',
  'Описание',
  'd00999_table_name',
  [
    { 
      name: 'column_name', 
      translation: 'Название столбца', 
      description: 'Описание', 
      dataType: 'string' 
    }
  ]
)
```

**Добавление связи:**
```javascript
new TableRelation('F00002', 'D00001', 'plan_date', 'calendar_date', 'many-to-one')
```

## 📚 Основные функции

### Каталог куба

| Функция | Описание |
|---------|----------|
| Выбор сущности | Выпадающий список с фактами (F) и измерениями (D) |
| Поиск | Фильтрация по названию, переводу, описанию |
| Показатели | Для фактов: название, перевод, описание, DAX формула |
| Атрибуты | Для измерений: столбец, перевод, описание, тип |
| Связи | Таблица со всеми связями и полями join |
| Data Lineage | Будущая интеграция с DBT/Git |

### Страница требований

| Функция | Описание |
|---------|----------|
| Создание требования | Название, описание, выбор группы |
| Добавление мер | С DAX выражениями и переводом |
| Добавление измерений | С описанием столбцов |
| Связи | Авто-связывание или ручное |
| Экспорт DOCX | Word документ с таблицами |
| Экспорт MD | Markdown для Git |
| Сохранение | В localStorage браузера |

## 🎨 Типы сущностей

### Факты (F-префикс)
- **Что:** Таблицы фактов с показателями
- **Пример:** F00002 (Планы), F00006 (Скидки)
- **Содержат:** Меры с DAX формулами
- **Отображаются как:** "Показатели"

### Измерения (D-префикс)
- **Что:** Справочники с атрибутами
- **Пример:** D00001 (Календарь), D00750 (SKU)
- **Содержат:** Столбцы с типами данных
- **Отображаются как:** "Атрибуты"

## ❓ Частые вопросы

**Q: Где хранятся данные?**
A: В localStorage браузера. Экспортируйте DOCX/MD для резервных копий.

**Q: Как добавить свой куб?**
A: Отредактируйте `src/mockData.js` - добавьте свои факты и измерения.

**Q: Что такое Data Lineage?**
A: Это будущая функция для отслеживания источников данных (DBT/Git интеграция).

**Q: Можно ли работать вдвоём?**
A: Нет, localStorage локальный. Используйте экспорт в Git для коллаборации.

**Q: Как обновить дизайн?**
A: Отредактируйте CSS файлы в `src/components/`.
